# Planner-Executor Agent - LangGraph Implementation

## Objective

Convert a working Planner-Executor agent from LangChain into a **LangGraph** workflow while preserving the same functionality, and fix correctness issues in the original implementation.

---

## Repository Structure

```
.
+-  main.py            # Entry point: configures LLM, MCP servers, and runs the graph
+-  graph.py           # LangGraph workflow: state, planner node, executor node, routing
```

## Requirements Implemented

### 1. State (graph.py)
```python
class AgentState(TypedDict):
    goal: str                        # user input / objective
    plan: List[Dict[str, Any]]       # steps generated by the planner
    current_step: int                # index of the step currently being executed
    results: List[Dict[str, Any]]    # outputs from all executed steps
```

### 2. Planner Node
- Takes the user goal, calls the LLM with a strict system prompt listing all available MCP tools and their exact argument names.
- Produces a structured JSON plan (list of steps with `tool` and `args`).

### 3. Executor Node
- Executes **one step per invocation**.
- If the step specifies a tool -> calls the MCP tool via `tools_map[tool_name].ainvoke(args)`.
- Tool results are normalised to plain text by `extract_text()` before being stored, preventing the LLM from receiving raw content-block dicts.
- If no tool -> calls the LLM for synthesis using a `SystemMessage` that forbids re-deriving numbers already computed by tools.

### 4. Graph Flow
```
START -> planner_node -> executor_node --+
                              ^          | (loop while steps remain)
                              +----------+
                                         | (all steps done)
                                        END
```

### 5. Tool Handling
- `safe_args()` remaps hallucinated argument names to the correct parameter before every tool call.
- `extract_text()` normalises MCP content-block responses to plain strings.
- Synthesis steps receive a strict `SystemMessage` so the LLM cannot invent or recalculate values the tools already produced.

## Available MCP Tools

| Tool | Server | Description |
|------|--------|-------------|
| `calculator(expression)` | math | Evaluate a math expression (ceil, sqrt, log, etc.) |
| `add / subtract / multiply / divide` | math | Basic arithmetic |
| `power / square_root` | math | Exponentiation and square root |
| `search_web(query)` | search | Web search via Tavily |
| `search_news(query)` | search | News search via Tavily |
| `get_current_weather(city)` | weather | Current weather (temp, humidity, wind) |
| `get_weather_forecast(city, days)` | weather | N-day weather forecast |

Supported cities: London, Paris, New York, Tokyo, Karachi, Lahore, Islamabad, Rawalpindi, Dubai, Berlin, Sydney, Chicago.

## How to Run

### 1. Start the weather server (keep this terminal open)
```bash
cd "d:\university\Agentic AI\Activity-1\Activity-1-main"
python Tools/weather_server.py
```

### 2. Run the main LangGraph agent
```bash
python main.py
```

### 3. To change the city or goal
Edit the `city` variable near the bottom of `main.py`:
```python
city = "New York"   # any supported city listed above
```

### 4. Run the standalone Planner-Executor (no LangGraph)
```bash
python Plan-Execu.py
```
